02

Issues Found

Critical

"Add playlists" on Board Detail has inverted UX flow

When a user clicks "Add playlists" on a board's detail page, SaveToBoardModal opens and asks them to pick a board — the user has to click the current board to add playlists to it. This is the opposite of what every user expects: pick playlists, then confirm. The component was designed for the "from playlist → choose board" direction and reused incorrectly here.

Fix: Create a dedicated AddPlaylistsToBoard modal that shows a checklist of playlists not yet in the current board. On confirm, call addPlaylistToBoard(boardId, playlistId) for each checked item. The SaveToBoardModal stays as-is for the playlist→board direction.
High

Discover page mood cards are non-functional

The "Explore moods" section renders 6 attractive gradient cards (Late Night Playlist, Feel-good Anthems, etc.) but clicking any of them does nothing. For a user who lands on Discover hoping to find music, this is a broken promise.

Fix: At minimum, clicking a mood card should navigate to the home page filtered by a matching label (if labels exist), or show a filtered subset of the user's playlists. Even a simple name-match filter ("chill", "focus") beats doing nothing.
High

Search bar doesn't filter results

The search input in the topbar accepts typing but the masonry grid doesn't update. The query is passed via outlet context but HomePage doesn't appear to consume it for filtering.

Fix: In HomePage, read the search query from useOutletContext() and filter the playlist array by name match before rendering the masonry grid. Debounce by 150ms.
Medium

Playlist detail track area shows no feedback on load failure

When the Spotify API call for tracks fails (expired token, network error), the track section below the hero is simply blank. There's no spinner, no error message, no explanation. Users will assume the playlist is empty.

Fix: Add an error state to the track fetch: catch the error and render a message like "Couldn't load tracks. Try again" with a retry button that re-runs the fetch.
Medium

Multi-select state doesn't clear when opening panels

Opening the Settings or Notifications panel while in multi-select mode leaves the selection action bar visible behind the panel, creating a visual collision and confusing state.

Fix: When a panel opens, call clearSelection() and exit selection mode. Or use z-index to ensure the panel fully covers the action bar and restore state on close.
Low

Sidebar icons lack tooltips on desktop

The fixed sidebar shows only icons. New users have no way to know what the grid icon, the wave icon, or the radio icon do without hovering. The current hover state is a background color change — not a label.

Fix: Add a positioned tooltip label that appears on hover (e.g., "Boards", "Radio", "Discover"). The icon already has title attributes — ensure they're visible as a real tooltip, not just the browser default.
03

UX Enhancement Suggestions

  • 1
    Add a playlist-picker to boards (separate from SaveToBoardModal)
    When on a board detail page, "Add playlists" should open a modal showing all playlists not already in the board — with checkboxes, search, and a single "Add" CTA. This is the most-used action on the board detail page and the current UX is backwards.
    Critical fix
  • 2
    Wire up Discover mood cards to filter or navigate
    Each mood prompt card should trigger something — filter by matching label, navigate to a smart board, or at minimum open a filtered search. Dead clicks destroy trust faster than any visual flaw.
    High impact
  • 3
    Live search filtering on the masonry grid
    The search input exists. Connect it. Filter playlists in real-time by name as the user types (debounced). Also filter board names and label names so a single search bar covers the whole library.
    High impact
  • 4
    Mobile-responsive layout pass
    The sidebar collapses correctly at narrow widths, but the masonry grid, modals, and detail pages need a mobile-first pass. At 375px the topbar search is too narrow and modals overflow. Given that a large segment of target users are mobile-primary, this is table stakes.
    Audience blocker
  • 5
    Board cover photo / custom mosaic
    Boards with playlists automatically get a 2×1 mosaic — good. But boards with 0 playlists show a gray rectangle. Let users upload a cover photo for any board (same pattern as playlist edit). Empty-state boards with no cover feel abandoned.
    Visual polish
  • 6
    Drag-to-reorder playlists within a board
    Once users add playlists to a board, they'll want to arrange them. The masonry layout makes drag-and-drop tricky, but even a drag handle on the card with snap-to-column ordering would be a significant UX upgrade for the "curators" persona.
    Power user
  • 7
    Onboarding flow for first-time users
    Users who connect Spotify and see a masonry grid of playlists don't know what to do next. A 3-step coach overlay — "this is your library", "create a board", "add playlists to it" — would dramatically reduce abandonment on first visit.
    Retention
  • 8
    Radio: basic shuffle playback before ElevenLabs
    The Radio page creates stations but they do nothing yet. Before ElevenLabs integration, implement a "Play station" button that opens Spotify with a shuffle of the selected playlists. That makes Radio feel real and valuable today.
    Feature gap
  • 9
    Label filter chips above the masonry grid
    Labels exist but there's no way to browse by label without going through the context menu per card. Add a horizontal chip row above the grid — click a label to filter the masonry to only playlists with that label. This surfaces the label system immediately.
    Discoverability
  • 10
    Keyboard shortcuts for power users
    Add: Cmd+K to focus search, Escape to close any modal/panel/selection, S to enter select mode on home. Power users will discover these and feel the app is polished. They're also fast to implement.
    Low effort, high signal

Maya — The Aesthetic Curator

24 · Graphic design student · Brooklyn, NY · 45+ playlists · 80% mobile

Pain points with Spotify

  • No visual library — just a flat list
  • No way to group by mood or season
  • Custom covers disappear or get overridden
  • Can't communicate a playlist's vibe at a glance

What resonates with Playlistify

  • Masonry grid immediately feels right
  • Boards map to her "dark academia", "summer 2023" mental model
  • Image upload in Edit modal
  • AI name suggestions (ironic and sincere)
  • Label system for granular tagging
📋

Daniel — The Library Organizer

33 · Product manager · Austin, TX · 60+ playlists · 50% desktop / 50% mobile

Pain points with Spotify

  • 60+ playlists in a flat list is unmanageable
  • No folder hierarchy in Spotify's native app
  • Liked Songs has 2,000-song graveyard
  • Can't quickly find the right playlist for a context

What resonates with Playlistify

  • Boards match his mental model: "Work", "Exercise", "Family"
  • Multi-select + bulk add to board for rapid organization
  • Archive for playlists he won't delete but doesn't use
  • Radio station concept for context-aware listening
📱

Zoe — The Social Listener

20 · College sophomore · Chicago, IL · 20 playlists · 98% mobile

Pain points with Spotify

  • Decision fatigue: can't pick what to play
  • Spotify home shows what she's heard, not what fits now
  • No way to signal vibe to herself or others
  • Friends' playlists feel more curated — comparison anxiety

What resonates with Playlistify

  • Discover page — she wants to be prompted
  • Visual masonry feels like Pinterest / TikTok — familiar
  • Mood boards match her social contexts
  • Labels for energy level and occasion
